++ Comparing Ruby classes & JavaScript constructor functions

Ruby Classes and Javascript constructors are objects we use in order to avoid redundancy in the code. ie. Repeating code over and over. Both are generic objects that contain generic properties, variables and methods/functions. In both we can create new instances of the objects and then assign the newly created objects specific properties like 'color' or numbers to do math to.

The differences are mostly syntactical and how the object is constructed. One big difference is the things the objects can 'do' are called 'Methods' in Ruby and 'Functions' in Javascript. I show the two styles below.

Ruby Class construct and new object:

Class Box

  def initialize(argument1, argument2)
   @variable_1 = argument1
   @variable_2 = argument2
  end

// now add a method which is something this object can 'do':

  def add_arguments
   @sum = @variable_1 + @variable_2
  end

end

make_a_box = Box.new(4,5)
make_a_box.add_arguments

Javascript object prototype construct and new object creation:

function Box(shape){

  this.shape = shape;
}

// now add a function which is something this object can 'do':

Box.prototype.tell_us_your_shape = function(){
  return this.shape;
}

var blueBox = new Box("cube");
blueBox.tell_us_your_shape();
