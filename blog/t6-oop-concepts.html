++ 3/17/05 Classes and Modules- Friend or foe or...what the H are they?

In this blog I will compare and explain classes and modules and what they are used for.

Classes

One of the key aspects in any programming is the idea of eliminating redundancy; finding repetitive code and finding a way to boil is all down to one logic train. This where classes come in.

A Class is an object like any object in the physical world; it has named attributes and actions it is capable of. These named attributes and actions are created by the programmer. These attributes are also non-specific in that we can create a copy of the class and at that time send in specific details like names, numbers, etc.

In this way a class eliminates repeating code as the class attributes are generic and the specifics are passed in at runtime per creation of a NEW copy of the class object.

Generic object construct:
class Person
  def initialize(persons_name,hair_color)
   @person = persons_name
   @hair_color = hair_color
  end
end

The class object is duplicated with specifics added:
a_person = Person.new("Bill","black")

Modules

Modules are similar to Classes in that they are generic encapsulations of functionality including methods and constants. Where they are different is you do not create new 'instances' of them. The Module is defined and then can be called or 'included' in a Class.

It is similar to Class inheritance except a Class can add more than one Module where Class inheritance can only inherit from one Class. Once the Module is called into the Class all the Modules methods, contants, etc can be used by the Class as if they were part of the Class.

Shown below is an example of the Module structure and how the Class calls it and uses its methods.

Generic Module construct:
module TryThisOut
     def do_math(number1,number2)
       number1 + number2
     end
end

Generic Class construct which calls the Module into the Class:
class Mod_test
  include TryThisOut
end

Now Create a new Class object and call the method the Module contained:
tester = Mod_test.new
p tester.do_math(6,4)
